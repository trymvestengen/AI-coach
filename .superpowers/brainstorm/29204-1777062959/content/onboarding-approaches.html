Onboarding-flyt: 3 tilnærminger

Velg strukturen som passer best

A

Full wizard (anbefalt)

Erstatter /register. Én skjerm per spørsmål: konto → mål → erfaring → frekvens → kropp → profilbilde. Slik Hevy gjør det. Fokusert, app-følelse.

1. Navn & konto → 2. Mål → 3. Erfaring → 4. Frekvens → 5. Kropp → 6. Bilde
B

Register → Onboarding

Behold /register for konto-opprettelse. Etter innlogging sendes nye brukere til /onboarding for profilspørsmål. Enklere å implementere, men to adskilte flyter.

Register (eksisterende) → Onboarding wizard → Home
C

Register → Én lang profil-skjerm

Hurtig registrering, deretter alle profilfelter på én lang scroll-skjerm. Raskest å implementere, men lite app-aktig og overveldende.

Register → Lang skjema-side → Home
